## 实现概述

### T4.2 事件巩固任务实现

**文件**: `backend/sbo_core/tasks_consolidation.py`

**功能实现**:
1. `consolidate_event(event_id)` 任务函数 - 使用 `@task_wrapper` 装饰器包装，支持重试机制（max_retries=3）
2. `InformationExtractor` 类 - 结构化信息抽取器，支持：
   - 实体识别（人物、时间、地点）
   - 关系识别（关联关系）
   - 偏好变化检测（含约束/禁忌）
   - 待办事项提取
   - 事实信息提取
   - 时间信息提取
3. 抽取结果写入 `extractions` 表
4. 任务完成后自动触发 `upsert_profile` 下游任务

**数据结构**:
- `ExtractionResult` - 抽取结果容器
- `ExtractedEntity` - 实体数据类
- `ExtractedRelation` - 关系数据类
- `ExtractedPreference` - 偏好数据类
- `ExtractedFact` - 事实数据类
- `ExtractedTodo` - 待办数据类

### T4.4 用户档案更新任务实现

**文件**: `backend/sbo_core/tasks_profile.py`

**功能实现**:
1. `upsert_profile(extraction_id)` 任务函数 - 支持重试机制（max_retries=3）
2. `ProfileConflictResolver` 类 - 冲突检测与解决：
   - 支持多种解决策略：OVERWRITE, VERSION, MERGE, REJECT
   - 根据字段类型自动选择策略
   - 稳定事实使用版本化策略
   - 约束字段强制使用版本化策略
3. `ProfileVersionManager` 类 - 版本化管理：
   - `archive_current_version()` - 归档当前版本到 `user_profile_versions` 表
   - `get_profile_history()` - 查询历史版本
4. 档案数据写入 `user_profiles` 表

## 关键链路验证

### 事件巩固链路
1. 原始事件录入 → 触发 `queue_consolidation_jobs` → `consolidate_event` 入队
2. Worker 执行 `consolidate_event` → 抽取结构化信息 → 写入 `extractions` 表
3. 检测到偏好/事实 → 触发 `upsert_profile` 入队

### 档案更新链路
1. `upsert_profile` 接收 `extraction_id` → 查询提取数据
2. 检测冲突 → 选择解决策略
3. 如需版本化 → 归档当前版本到 `user_profile_versions`
4. 更新 `user_profiles` 表，版本号递增
