## 问题清单

### 🔴 阻断性问题（0）

无阻断性问题。

### 🟠 严重问题（2）

#### 1. 缺少 Cross-Encoder 重排实现
- **位置**：`backend/sbo_core/retrieval_pipeline.py:_optional_rerank`
- **原因**：任务 3.4.3 要求实现 cross-encoder 重排，但当前实现仅为模拟（使用随机扰动）
- **影响**：Deep 模式检索质量无法达到设计要求，无法提供真实的重排能力
- **建议**：
  - 实现真实的 cross-encoder 调用（对接外部 rerank provider）
  - 或在文档中明确标注为"阶段 1 简化实现"，并创建后续任务跟踪
  - 确保降级策略（rerank 失败回退到 fusion）已正确实现

#### 2. 缺少生命周期/衰减的访问计数更新机制
- **位置**：`backend/sbo_core/retrieval_pipeline.py`
- **原因**：设计文档要求"访问计数/最后访问时间的更新必须通过明确写路径（例如 `/query` 返回后异步记录）"
- **影响**：无法实现阶段 2 的 access_count/last_accessed_at 强化衰减
- **建议**：
  - 在 `query_service.py` 中添加异步更新访问计数的逻辑
  - 或在文档中明确标注为"阶段 2 实现"，当前仅实现 time-decay

### 🟡 一般问题（5）

#### 1. 硬编码的默认用户 ID
- **位置**：`backend/sbo_core/routes/ingest.py:chat_interaction`、`backend/sbo_core/routes/manage.py`
- **原因**：使用硬编码的 `user_id = "default_user"` 或 `_DEFAULT_USER_ID`
- **影响**：单机单用户形态可接受，但未来扩展为多用户时需要重构
- **建议**：添加 TODO 注释说明未来需要从认证信息中获取真实用户 ID

#### 2. LLM 调用未实现
- **位置**：`backend/sbo_core/routes/ingest.py:chat_interaction`
- **原因**：LLM 生成回复部分仅返回占位符文本
- **影响**：`/chat` 接口无法提供真实的对话能力
- **建议**：
  - 实现 LLM 调用逻辑（对接 llama.cpp 或外部 provider）
  - 或在文档中明确标注为"阶段 1 简化实现"

#### 3. 文件存储未实现
- **位置**：`backend/sbo_core/routes/ingest.py:upload_file`
- **原因**：文件上传后未实际保存到存储系统
- **影响**：文件上传功能不完整，无法实际使用
- **建议**：
  - 实现文件存储逻辑（本地文件系统或对象存储）
  - 或在文档中明确标注为"阶段 1 简化实现"

#### 4. 检索答案提示生成过于简单
- **位置**：`backend/sbo_core/query_service.py:_generate_answer_hint`
- **原因**：答案提示生成仅拼接前两条证据文本，未使用 LLM
- **影响**：答案提示质量较低，用户体验不佳
- **建议**：实现基于 LLM 的答案提示生成

#### 5. 语义相似度计算过于简化
- **位置**：`backend/sbo_core/retrieval_pipeline.py:_calculate_semantic_similarity`
- **原因**：使用 Jaccard 相似度 + 随机性模拟向量相似度
- **影响**：检索质量无法达到生产级别
- **建议**：
  - 实现真实的向量相似度计算（使用 embeddings）
  - 或在文档中明确标注为"阶段 1 简化实现"

### 🟢 优化建议（3）

#### 1. 缓存策略可优化
- **位置**：`backend/sbo_core/query_service.py:_query_cache`
- **原因**：使用简单的字典缓存，无 LRU 淘汰策略
- **建议**：考虑使用 `functools.lru_cache` 或 Redis 缓存

#### 2. 错误日志可增强
- **位置**：多处 `except Exception` 块
- **原因**：部分错误处理仅记录简单日志
- **建议**：增加更多上下文信息（如请求参数、用户 ID 等）

#### 3. 测试覆盖可增强
- **位置**：`backend/tests/`
- **原因**：部分边界条件和错误路径测试覆盖不足
- **建议**：增加更多边界条件测试（如并发、超时、大数据量等）
